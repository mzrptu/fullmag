## 3. Model fizyczny, który powinien być celem

## 3.1. Równanie ruchu

Podstawą jest LLG z dodatkowymi wkładami od STT, pola Oersteda i ewentualnie temperatury:

$$
\frac{\partial \mathbf m}{\partial t}
=
-\gamma \mu_0 \, \mathbf m \times \mathbf H_\mathrm{eff}
+
\alpha \, \mathbf m \times \frac{\partial \mathbf m}{\partial t}
+
\boldsymbol{\tau}_\mathrm{STT}
+
\boldsymbol{\tau}_\mathrm{th}.
$$

gdzie

$$
\mathbf H_\mathrm{eff}
=
\mathbf H_\mathrm{ex}
+
\mathbf H_\mathrm{demag}
+
\mathbf H_\mathrm{ext}
+
\mathbf H_\mathrm{ani}
+
\mathbf H_\mathrm{DMI}
+
\mathbf H_\mathrm{mel}
+
\mathbf H_\mathrm{Oe}.
$$

W praktyce dla Twojego przypadku kluczowe będą:

- `exchange`
- `demag`
- `uniaxial anisotropy` / efektywna anizotropia CoFeB
- `Oersted`
- `Slonczewski STT`
- opcjonalnie `thermal noise`

---

## 3.2. Pole Oersteda od nanopillara

Dla idealizowanego długiego przewodnika cylindrycznego:

$$
H_\phi(r)=
\begin{cases}
\frac{I\, r}{2\pi R^2}, & r < R \\
\frac{I}{2\pi r}, & r \ge R
\end{cases}
$$

gdzie:

- $I$ — prąd,
- $R$ — promień przewodnika,
- kierunek pola jest obwodowy.

To jest bardzo naturalny mechanizm pobudzenia vortexu, bo:
- dobrze sprzęga się z ruchem rdzenia,
- może selektywnie wzmacniać/hamować określoną chiralność,
- przy pobudzeniu AC daje dostęp do rezonansu gyrotropowego i modów wyższych.

---

## 3.3. Spin-transfer torque

### Zhang–Li
Ogólna postać ciągła wygląda typowo tak:

$$
\boldsymbol{\tau}_\mathrm{ZL}
=
-(\mathbf u \cdot \nabla)\mathbf m
+
\beta \, \mathbf m \times (\mathbf u \cdot \nabla)\mathbf m.
$$

Ta postać jest najbardziej naturalna dla **CIP** i tekstur przestrzennych.

### Slonczewski
Dla **CPP MTJ** ważniejsza jest postać typu Slonczewski:

$$
\boldsymbol{\tau}_\mathrm{Slonc}
=
\sigma(J, P, \Lambda, d, M_s, \ldots)\;
\mathbf m \times (\mathbf m \times \mathbf p)
+
\sigma'(J, \epsilon', \ldots)\;
\mathbf m \times \mathbf p.
$$

gdzie:

- $\mathbf p$ — kierunek polaryzacji spinowej (od warstwy referencyjnej),
- $J$ — gęstość prądu,
- $\Lambda$ i $\epsilon'$ — parametry modelu,
- pierwszy człon to zwykle torque damping-like,
- drugi człon to field-like correction.

## 3.4. Szum termiczny

Dla termicznego pola Browna typowy wariant ma wariancję:

$$
\langle H_i^\mathrm{th}(t) H_j^\mathrm{th}(t') \rangle
\propto
\frac{2 \alpha k_B T}{\gamma \mu_0 M_s V \Delta t}
\delta_{ij}\delta_{tt'}.
$$

W praktyce:
- bez temperatury da się policzyć próg i częstotliwość nominalną,
- ale **linewidth** i stabilność fazowa wymagają już modelu termicznego.

---

## 3.5. Redukcja Thiele’a jako narzędzie diagnostyczne

Dla vortexu bardzo użyteczny jest opis rdzenia przez położenie $\mathbf X = (X,Y)$:

$$
\mathbf G \times \dot{\mathbf X}
-
D \dot{\mathbf X}
-
\nabla U(\mathbf X)
+
\mathbf F_\mathrm{drive}
=
0.
$$

To nie zastępuje pełnej mikromagnetyki, ale jest bardzo dobre do:

- interpretacji częstotliwości gyrotropowej,
- interpretacji promienia orbity,
- wykrywania wejścia w autooscylację,
- projektowania obserwabli GUI.

## 4. Jak modelować dokładnie Twój układ

## 4.1. CoFeB(1 nm)/NiFe(8 nm): trzy poziomy dokładności

### Poziom A — efektywny free layer
Jeden cylinder:

- średnica `400 nm`
- grubość `9 nm`
- skuteczne:
  - `Ms_eff`
  - `A_eff`
  - `alpha_eff`
  - `Ku_eff`

To jest najlepszy **pierwszy krok**.

Zalety:
- szybkość
- prostota
- łatwo walidować vortex i $f_g$

Wady:
- nie rozdzielasz wpływu cienkiego CoFeB od grubego NiFe

### Poziom B — jawna niejednorodność po `z`
Jedna geometria, ale pola materiałowe zależne od `z`:

- `0–1 nm` → CoFeB
- `1–9 nm` → NiFe

To jest moim zdaniem najlepsza droga do realistycznego modelu przy obecnym stanie repo.

### Poziom C — pełny stos MTJ
Warstwa swobodna + referencyjna + bariera + sprzężenie transportowe.

Tego **nie rekomenduję jako pierwszy etap**, bo obecna semantyka repo nie jest jeszcze na to gotowa.

## 4.2. Dyskretyzacja

Dla vortexu 400 nm warto zacząć od:

- `dx = dy = 2–4 nm`
- `dz = 1 nm`

Jeżeli chcesz jawnie odróżnić `CoFeB(1 nm)`:
- `dz = 1 nm` to sensowne minimum,
- `dz = 0.5–1 nm` jeśli chcesz lepiej traktować bardzo cienką warstwę.

Praktyczny kompromis:

```text
wariant szybki:   (4 nm, 4 nm, 1 nm)
wariant produkcyjny: (2.5 nm, 2.5 nm, 1 nm)
wariant dokładniejszy po z: (2–2.5 nm, 2–2.5 nm, 0.5–1 nm)
```

## Wniosek wykonawczy

Dla właściwych przebiegów time-domain STNO preferowałbym:
- **FDM**
- **native CUDA**
- double precision, jeśli koszt akceptowalny

CPU reference zostawiłbym:
- do testów regresyjnych,
- do małych przypadków kontrolnych,
- nie jako główny silnik dużego dysku 400 nm z długimi przebiegami.

### Etap 2 — słabe pobudzenie Oerstedem

- linewidth

## 5. Co jest już publicznie dostępne teraz

## 5.1. Przykład: relaksacja vortexu i pobudzenie polem Oersteda

Poniżej przykład zgodny z aktualnym idiomem klas repo (ścieżki importu mogą się jeszcze minimalnie różnić między branchami, ale nazwy klas i pól odpowiadają stanowi repo):

```python
from fullmag.init import uniform
from fullmag.model.dynamics import LLG
from fullmag.model.discretization import DiscretizationHints, FDM
from fullmag.model.energy import Exchange, Demag, Zeeman, OerstedCylinder, Sinusoidal
from fullmag.model.geometry import Cylinder
from fullmag.model.outputs import SaveScalar, Snapshot
from fullmag.model.problem import (
    Problem,
    RuntimeSelection,
    BackendTarget,
    DeviceTarget,
    ExecutionMode,
)
from fullmag.model.structure import Material, Ferromagnet
from fullmag.model.study import Relaxation, TimeEvolution

disk = Cylinder(radius=200e-9, height=9e-9, name="free_disk")

mat = Material(
    name="free",
    Ms=8.0e5,
    A=13e-12,
    alpha=0.01,
)

free = Ferromagnet(
    name="free",
    geometry=disk,
    material=mat,
    m0=uniform((1.0, 0.0, 0.0)),
)

runtime = RuntimeSelection(
    backend_target=BackendTarget.FDM,
    device_target=DeviceTarget.CUDA,
    execution_mode=ExecutionMode.EXTENDED,
)

disc = DiscretizationHints(
    fdm=FDM(default_cell=(2.5e-9, 2.5e-9, 1.0e-9))
)

relax_problem = Problem(
    name="vortex_relax",
    magnets=[free],
    energy=[
        Exchange(),
        Demag(),
        Zeeman((0.0, 0.0, 0.0)),
    ],
    study=Relaxation(
        outputs=[
            SaveScalar("time", every=1e-11),
            SaveScalar("max_h_eff", every=1e-11),
            Snapshot("m", "z", every=2e-10),
        ],
        algorithm="llg_overdamped",
        max_steps=50000,
        dynamics=LLG(integrator="heun", fixed_timestep=5e-14),
    ),
    discretization=disc,
    runtime=runtime,
)

drive_problem = Problem(
    name="vortex_oersted_drive",
    magnets=[free],
    energy=[
        Exchange(),
        Demag(),
        OerstedCylinder(
            current=4e-3,             # 4 mA
            radius=200e-9,
            time_dependence=Sinusoidal(
                frequency_hz=350e6,
                phase_rad=0.0,
                offset=0.0,
            ),
        ),
    ],
    study=TimeEvolution(
        dynamics=LLG(integrator="heun", fixed_timestep=2e-14),
        outputs=[
            SaveScalar("time", every=2e-12),
            SaveScalar("mx", every=2e-12),
            SaveScalar("my", every=2e-12),
            SaveScalar("mz", every=2e-12),
            Snapshot("m", "z", every=1e-10),
        ],
    ),
    discretization=disc,
    runtime=runtime,
)
```

### Co ten przykład daje realnie
- relaksację do vortexu,
- czasową odpowiedź na AC Oersteda,
- podstawę do FFT z `mx(t), my(t), mz(t)`

### Czego jeszcze nie daje
- publicznego STT
- temperatury w jawnej, pewnej ścieżce publicznej
- gotowego GUI do analizy widmowej

## 5.2. Przykład: publiczny tor RF-antenowy dla fal spinowych

```python
from fullmag.model.antenna import (
    AntennaFieldSource,
    MicrostripAntenna,
    RfDrive,

from fullmag.model.outputs import SaveSpectrum, SaveDispersion, SaveEigenDiagnostics
from fullmag.model.study import FrequencyResponse

strip = AntennaFieldSource(
    name="rf_strip",
    antenna=MicrostripAntenna(

        thickness=100e-9,
        height_above_magnet=50e-9,
        preview_length=4e-6,
        center_x=0.0,

    drive=RfDrive(
        current_a=5e-3,
        frequency_hz=8e9,
        phase_rad=0.0,
    ),
)

fr_problem = Problem(
    name="spin_wave_excitation",
    magnets=[free],
    energy=[Exchange(), Demag()],
    current_modules=[strip],
    excitation_analysis=SpinWaveExcitationAnalysis(
        source="rf_strip",
        propagation_axis=(1.0, 0.0, 0.0),
        k_max_rad_per_m=8e7,
        samples=256,
    ),
    study=FrequencyResponse(
        outputs=[
            SaveSpectrum(),
            SaveDispersion(),
            SaveEigenDiagnostics(),

        frequencies_hz=[1e9, 2e9, 3e9, 4e9, 5e9],
        excitation_field_au_per_m=(0.0, 1.0, 0.0),
    ),
    discretization=disc,
    runtime=runtime,
)
```

## 9.2. Jak liczyć pozycję rdzenia

Najprostszy wariant roboczy:

$$
X_c = \frac{\int x \, w(\mathbf r)\, dA}{\int w(\mathbf r)\, dA}, \quad
Y_c = \frac{\int y \, w(\mathbf r)\, dA}{\int w(\mathbf r)\, dA}
$$

z wagą typu:

$$
w(\mathbf r) = \max(m_z(\mathbf r), 0)^p
$$

dla odpowiednio dobranego `p` (np. `p=2...6`) albo przez lokalne maksimum `|m_z|` w obszarze rdzenia.

Wersja dokładniejsza:
- użyć gęstości topologicznej
- albo detekcji maksimum `|q(\mathbf r)|`

ale na MVP prostszy tracker jest w zupełności wystarczający.

## 9.3. Jak liczyć widmo

Najprostsza ścieżka post-process:

```python
import numpy as np

def psd_from_trace(t, x):
    t = np.asarray(t)
    x = np.asarray(x)
    dt = float(np.mean(np.diff(t)))
    x = x - np.mean(x)

    window = np.hanning(len(x))
    X = np.fft.rfft(x * window)
    f = np.fft.rfftfreq(len(x), dt)
    S = (np.abs(X) ** 2) / np.sum(window ** 2)
    return f, S
```

To można dziś robić **poza GUI** już teraz na `mx/my/mz`.

Docelowo GUI powinno:
- umieć wybrać kanał,
- odrzucić transjent,
- zrobić okno,
- pokazać peak,
- dopasować linewidth.

## 10. Czego nie warto robić w pierwszym kroku

## 10.1. Nie zaczynać od pełnego stacku MTJ

Nie zaczynałbym od:
- osobnej warstwy referencyjnej,
- jawnego tunelowania,
- modelu `R(t)` / `V(t)`,
- bias-dependent polarization

To za wcześnie.

## 10.2. Nie zaczynać od eigen-solvera jako walidatora vortexu

Dla vortexu główny workflow powinien być:

```text
relax -> time drive -> trace -> FFT -> orbit
```

a nie:

```text
relax -> eigenmodes -> uznajemy, że to wszystko
```

## 10.3. Nie mieszać od razu Oersteda, STT i temperatury

Najpierw:
1. vortex
2. Oersted
3. STT
4. temperatura

To bardzo zmniejsza ryzyko błędnej diagnozy.

## 15. Przykład docelowego skryptu po wdrożeniu braków

To jest przykład tego, jak chciałbym widzieć API po minimalnym rozszerzeniu `Problem`:

```python
from fullmag.init import uniform
from fullmag.model.dynamics import LLG
from fullmag.model.discretization import DiscretizationHints, FDM
from fullmag.model.energy import Exchange, Demag, OerstedCylinder, Constant, UniaxialAnisotropy
from fullmag.model.geometry import Cylinder
from fullmag.model.outputs import SaveScalar, Snapshot
from fullmag.model.problem import (
    Problem,
    RuntimeSelection,
    BackendTarget,
    DeviceTarget,
    ExecutionMode,
)
from fullmag.model.structure import Material, Ferromagnet
from fullmag.model.study import TimeEvolution

disk = Cylinder(radius=200e-9, height=9e-9, name="free_disk")

mat = Material(
    name="free",
    Ms=8.0e5,
    A=13e-12,
    alpha=0.015,
    Ku1=2.0e4,
    anisU=(0.0, 0.0, 1.0),
)

free = Ferromagnet(
    name="free",
    geometry=disk,
    material=mat,
    m0=uniform((1.0, 0.0, 0.0)),
)

problem = Problem(
    name="cpp_mtj_vortex_stno",
    magnets=[free],
    energy=[
        Exchange(),
        Demag(),
        UniaxialAnisotropy(ku1=2.0e4, axis=(0.0, 0.0, 1.0)),
        OerstedCylinder(
            current=6e-3,
            radius=200e-9,
            time_dependence=Constant(),
        ),
    ],
    study=TimeEvolution(
        dynamics=LLG(integrator="heun", fixed_timestep=2e-14),
        outputs=[
            SaveScalar("time", every=2e-12),
            SaveScalar("mx", every=2e-12),
            SaveScalar("my", every=2e-12),
            SaveScalar("mz", every=2e-12),
            Snapshot("m", "z", every=5e-11),
        ],
    ),
    current_density=(0.0, 0.0, 8e10),
    stt_spin_polarization=(0.0, 0.0, 1.0),
    stt_degree=0.45,
    stt_lambda=2.0,
    stt_epsilon_prime=0.0,
    temperature=300.0,
    discretization=DiscretizationHints(
        fdm=FDM(default_cell=(2.5e-9, 2.5e-9, 1.0e-9))
    ),
    runtime=RuntimeSelection(
        backend_target=BackendTarget.FDM,
        device_target=DeviceTarget.CUDA,
        execution_mode=ExecutionMode.EXTENDED,
    ),
)
```

To jest dokładnie ten poziom ergonomii, który odblokuje Ci realną pracę.
